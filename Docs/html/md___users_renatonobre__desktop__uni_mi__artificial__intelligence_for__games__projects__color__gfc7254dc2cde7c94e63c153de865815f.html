Genetic Algorithm

All files related to the genetic algorithm process is in the Genetic Algorithm folder. The Simulation class handles the whole population and the Genetics handles the algorithm flow with a focus on the individuals, while other files serve as supporting.

Theorical Basis

Genetic Algorithms (GAs) are a particular class of algorithms applied to solve many classes of problems, mainly belonging to the Artificial Intelligence (AI) field, but – more in general – they are useful in many optimization problems and in heuristic search processes [1].

For the purpose of this work the Genetic Algorithm will not be used to generate an optimization problem, neither an heuristic process. Instead, the algorithm will be used to create variation of the population. Representing multiple mobs in a MMO environment.

A tipical genetic algorith, works in a process as described below:

First a initial population is created based on a certain rule defined by the user. This population is then evaulated in its fitness criterea, which will evaluate each of the members of the population if they are suitable for the environment. The selected individuals will be interated to the reproduction process. To produce “children” the algorithm operates genetic recombination techniques (crossover) and mutations on chromosomes represented by bit sequences. When the new generation is created it will evalute if the population is the desired one, iterating until some optimal solution is reached.

For this project the algorithm iterates for as long as we want. Allowing the evaluation of the created population.

GA's Steps on the Color Golems Project

Initial Population

This project map is randomly generated in a given size. Once the map is generated, six different zones will be created. Three zones for primary collors: red, yellow and blue; and three zones for secondary colors: green, purple and orange.

The population size will be equal to the defined population max size.

Selection

The selection process will be created using the DeltaE2000 color diference score. Will be taken in consideration the diference of the color for the Golem.

Here the selection will not necessarly kill any Golem. Golems have health points, the bigger the diference the more damage it gets in each simulation tick. Although some golems will die due to unsuitability to the environment, it is necessary to keep a control on the population numerosity, in order to avoid overpopulation [1]. This implies defining an upper bound to the number of individuals. Therfore the golems also have an age parameter, indicating how long this golem is able to survive.

Crossover

The Crossover between Golems will happen as described by the GOLEM paper, with one of possible three methods for selection:

  1. Single point crossover: the chromosome is split into two parts in a randomly selected point. The chromosome describing the offspring is composed by the first section of the first parent chromosome and by the second part of the second parent chromosome.
  2. Two points crossover: the chromosome is split into three parts, the first and third part of the first parent plus the second part of the second parent become the genetic heritage of the first whelp;
  3. Uniform crossover: this approach provides a higher genetic variation, since each gene of the whelp is copied – randomly – from one of the corresponding gene belonging to one of the parents.

To allow the creation of 1 to many childrens, the crossover is random for each of the childrens. This means if that the crossover is to produce two childrens, the second one will not be the complement of the first, it will also break on a random point.

Parents will be selected based on the distance from one to another. Biological sex will be disconsidered for the mating process.

Childrens will spawn on the midpoint of two parents.

Mutation

Mutations will happen in a random probabilty, the mutation will interfere in the Golem Chromosome in multiple possible ways, changing its possible variations.

Termination Criteria

The simulation termination criteria will be considered as a maximum generation value, which could also be infinity if we would like to control the simulation. Also the player might stop/play or reset the simulation at any time.
